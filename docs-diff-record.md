### 与项目相关的关键发现

#### 1.1 标识符 63 字符限制
- **文档说明**: Harbour 运行时标识符只有前 63 个字符是有效的（不区分大小写）
- **项目影响**: 需要检查项目中的变量名、函数名是否超过 63 字符
- **冲突检测**:
  - `LOCAL` 变量：编译时检测重复
  - `STATIC` 变量：编译时检测重复
  - `FUNCTION/PROCEDURE`：编译时检测重复
  - `CLASS` 名称：编译时检测重复
  - **注意**: `METHOD`、`DATA`、`VAR`、`PUBLIC/PRIVATE` **不检测重复**，后定义会覆盖前定义

#### 1.2 严格相等 vs 宽松相等
- **文档说明**:
  - `==`: 严格相等，比较字符串长度和每个字符，包括尾随空格
  - `=`: 宽松相等，受 `SET EXACT` 影响
- **项目建议**: 在字符串比较时始终使用 `==` 以避免错误
- **当前项目**: 需要检查代码中的字符串比较是否使用了 `=`

#### 1.3 赋值运算符
- **文档说明**:
  - `:=`: 规范的赋值运算符
  - `=`: 重载运算符，既可赋值也可比较（容易混淆）
- **项目建议**: 始终使用 `:=` 进行赋值
- **当前项目**: 项目代码已经遵循此规范

#### 1.4 字符串字面量
- **文档说明**: 支持四种形式：
  - `"abc"` - 标准双引号
  - `'abc'` - 标准单引号
  - `[abc]` - 方括号（不能包含 `]` 字符）
  - `e"line\nnext"` - C 风格转义字符串
- **项目建议**: 需要换行或转义时使用 `e"..."`

#### 1.5 UTF-8 编码支持
- **文档说明**:
  - 推荐使用 `UTF8EX` 代码页（而非 `UTF8`）
  - 在 Main() 中请求并选择 UTF8EX:
    ```harbour
    REQUEST HB_CODEPAGE_UTF8EX
    FUNCTION Main()
       hb_cdpSelect( "UTF8EX" )
    ```
- **当前项目**: 需要检查是否已正确设置 UTF8EX

#### 1.6 数组初始化警告
- **文档说明**: `Array(10, 9)` 可能创建错误结构
- **项目影响**: 这与 AGENTS.md 中的警告一致
- **推荐**: 手动创建二维数组

#### 1.7 换行符处理
- **文档说明**: 支持 `\n`、`\r`、`\r\n`
- **推荐使用**: `hb_eol()` 函数获取操作系统合适的换行符
- **当前项目**: 记忆中有使用 `OutErr("调试信息", hb_eol())` 的规范

## 2. 个人编码标准 (personal_coding_standards.md)

### 与现有项目的关系
这是一个跨项目的编码标准文档，定义了保持代码一致性和可读性的规范。

### 主要规范

#### 2.1 缩进和空白
- **规范**: 4 空格缩进，不允许制表符
- **特殊规则**: 函数/过程/方法体内的初始语句不缩进（从第 1 列开始）
- **当前项目**: 需要检查项目代码是否遵循此规范

#### 2.2 行首不加空白
- **规范**: 不要在 function/procedure/method 声明行后添加空行
- **规范**: 不要在 for 循环头行后添加空行

#### 2.3 命名规范（匈牙利命名法）

**局部变量**:
- 格式: `l_` + 类型字母 + CamelCase
- 类型字母:
  - `c` = 字符串
  - `n` = 数值
  - `i` = 整数
  - `d` = 日期
  - `t` = 时间戳
  - `l` = 逻辑
  - `a` = 数组
  - `h` = 哈希
  - `o` = 对象
  - `p` = 指针
  - `x` = 未知/可变类型
- 示例: `l_cCustomerName`, `l_nTotalAmount`, `l_aRecordList`

**参数**:
- 格式: `par_` + 类型字母 + CamelCase
- 示例: `par_cCustomerName`, `par_nTotalAmount`

**预处理器符号**:
- 格式: 全大写，用 `_` 分隔
- 示例: `MIN_HARBOUR_ORM_VERSION`

#### 2.4 Harbour 关键字大小写
- **规范**: 所有 Harbour 语言关键字必须**小写**
- 包括: `function`, `local`, `if`, `elseif`, `endif`, `return`, `class`, `method` 等
- **当前项目**: 需要检查是否遵循

#### 2.5 赋值运算符
- **规范**: 所有变量赋值使用 `:=`（绝不使用 `=`）
- **当前项目**: 已遵循此规范

#### 2.6 字符串拼接
- **规范**: 追加到现有变量时使用 `+=`
- 示例:
  ```harbour
  l_cHtml += "<div>"
  l_cHtml += "</div>"
  ```

#### 2.7 一行一个 LOCAL 声明
- **规范**: 避免在一行定义多个变量
- **推荐**: 每个变量占一行，方便添加注释

#### 2.8 函数/过程分隔符
- **规范**: 在函数/过程/方法之间放置分隔线
- 样式 1（默认）: `//` + 80 个 `-`
  ```harbour
  //--------------------------------------------------------------------------------
  ```
- 样式 2（可读性优先）: `//` + 100 个 `=`
  ```harbour
  //====================================================================================================
  ```
- **规范**: 不要在 return 语句和分隔线之间插入空行

#### 2.9 类分隔符
- **规范**: 类之间放置分隔线（80+ 个 `=`）

#### 2.10 注释
- **规范**: 使用 `//` 注释
- **目的**: 解释**为什么**，而不是代码显然做了**什么**

#### 2.11 控制流偏好
- **规范**: 深层嵌套的可接受
- **规范**: 避免仅为了扁平化逻辑而引入临时变量
- **规范**: 仅在多处使用时才创建辅助函数
- **规范**: 长函数可以接受，只要服务单一目的
- **规范**: 优先使用单退出例程，避免中途 return

#### 2.12 显式 END 形式（Harbour Extended Library）
- **规范**: 虽然允许通用 `end`，但优先使用显式终止符:
  - `endtry`
  - `enddo`
  - `endfor`
  - `endcase`
  - `endwith`
  - `endif`
  - `endswitch`
  - `endwhile`
- **大小写规则**: 在 Harbour 源代码中**始终小写**

#### 2.13 iif() vs if()
- **规范**: 优先使用 `iif()` 以避免与 `if/elseif/else/endif` 语句语法的视觉混淆
