本文档记录了从 `docs/` 目录采纳的 Harbour 语言编码规范，适用于中国象棋 GUI 项目。

## 1. 命名规范

### 1.1 局部变量 (LOCAL)
- **格式**: `l_` + 类型字母 + CamelCase
- **类型字母**:
  - `c` = 字符串 (character/string)
  - `n` = 数值 (numeric)
  - `i` = 整数 (integer，明确表示整数类型)
  - `d` = 日期 (date)
  - `t` = 时间戳 (timestamp)
  - `l` = 逻辑/布尔 (logical)
  - `a` = 数组 (array)
  - `h` = 哈希/映射 (hash)
  - `o` = 对象 (object)
  - `p` = 指针/句柄 (pointer，外部资源句柄)
  - `x` = 未知/可变类型 (unknown)

**示例**:
```harbour
local l_cCustomerName := ""
local l_nTotalAmount  := 0
local l_lIsEnabled    := .f.
local l_aRecordList   := {}
local l_hRowByName    := {=>}
local l_oConnection   := nil
local l_xResult       := nil
```

### 1.2 参数 (PARAMETERS)
- **格式**: `par_` + 类型字母 + CamelCase

**示例**:
```harbour
par_cCustomerName
par_nTotalAmount
par_lIsEnabled
par_xResult
par_pPyObject
```

### 1.3 预处理器符号 (#define)
- **格式**: 全大写，用 `_` 分隔单词

**示例**:
```harbour
#define MIN_HARBOUR_VERSION      "3.2.0"
#define MAX_AI_THINK_TIME        30000
#define BOARD_ROWS               10
#define BOARD_COLS               9
```

### 1.4 标识符长度限制
- **有效长度**: 前 63 个字符（不区分大小写）
- **建议**: 标识符长度 ≤ 40 字符以避免冲突
- **碰撞检测**:
  - ✅ `LOCAL`, `STATIC`, `FUNCTION`, `CLASS` - 编译时检测重复
  - ⚠️ `METHOD`, `DATA`, `VAR`, `PUBLIC`, `PRIVATE` - **不检测重复**，后定义覆盖前定义

**警告**:
```harbour
// 危险！这两个会被视为同一个标识符
local VeryLongVariableNameThatExceedsSixtyThreeCharacters_ABC := 1
local VeryLongVariableNameThatExceedsSixtyThreeCharacters_XYZ := 2  // 覆盖上面的值
```

## 2. 代码风格

### 2.1 关键字大小写
- **规则**: 所有 Harbour 语言关键字必须**小写**

**正确**:
```harbour
function Main()
   local l_nValue := 0
   if l_nValue > 0
      l_nValue += 1
   endif
return nil
```

**错误**:
```harbour
FUNCTION Main()
   LOCAL l_nValue := 0
   IF l_nValue > 0
      l_nValue += 1
   ENDIF
RETURN NIL
```

### 2.2 赋值运算符
- **规则**: 始终使用 `:=` 进行赋值
- **禁止**: 使用 `=` 赋值（易与相等比较混淆）

**正确**:
```harbour
local l_nValue := 10
l_nValue := l_nValue + 5
```

**错误**:
```harbour
local l_nValue = 10   // 虽然编译通过，但不推荐
```

### 2.3 字符串比较
- **规则**: 字符串比较使用 `==`（严格相等）
- **避免**: 使用 `=`（宽松相等，受 `SET EXACT` 影响）

**正确**:
```harbour
if l_cText == "Hello"
   // ...
endif
```

**错误**:
```harbour
if l_cText = "Hello"   // 可能匹配 "Hello ", "HelloX" 等
   // ...
endif
```

### 2.4 字符串拼接
- **规则**: 追加到现有变量时使用 `+=`

**示例**:
```harbour
local l_cHtml := ""
l_cHtml += "<div>"
l_cHtml += "内容"
l_cHtml += "</div>"
```

### 2.5 缩进和空白
- **缩进**: 4 个空格，**禁止使用制表符**
- **函数体**: 初始语句**不缩进**（从第 1 列开始），仅嵌套块缩进
- **空行**:
  - 不要在 `function`/`procedure`/`method` 声明行后添加空行
  - 不要在 `for` 循环头行后添加空行
  - 不要在 `return` 语句和分隔线之间添加空行

**示例**:
```harbour
function TestFunction()
   local l_nValue := 0

   for l_nValue := 1 to 10
      if l_nValue == 5
         exit
      endif
      ? l_nValue
   next
return nil

//--------------------------------------------------------------------------------
```

### 2.6 注释风格
- **规则**: 使用 `//` 单行注释
- **目的**: 解释**为什么**（why），而非代码显然做了**什么**（what）

**示例**:
```harbour
// 检查将军状态，因为将军后需要特殊处理
if xq_IsKingInCheck(l_aBoard, l_lRedTurn)
   l_lInCheck := .t.
endif
```

### 2.7 函数分隔符
- **位置**: 在函数/过程/方法之间
- **样式**: `//` + 80 个 `-`
- **规则**: 分隔线前不添加空行

**示例**:
```harbour
function Function1()
   // ...
return nil

//--------------------------------------------------------------------------------

function Function2()
   // ...
return nil
```

### 2.8 变量声明
- **规则**: 一行一个 `LOCAL`/`STATIC` 声明
- **目的**: 便于添加注释，保持 diff 清晰

**正确**:
```harbour
local l_cFirstName := ""      // 名字
local l_cLastName  := ""      // 姓氏
local l_nAge       := 0       // 年龄
```

**错误**:
```harbour
local l_cFirstName := "", l_cLastName := "", l_nAge := 0
```

## 3. UTF-8 编码规范

### 3.1 代码页设置
- **规则**: 始终使用 UTF8EX 代码页
- **位置**: 在 `Main()` 函数开头

**必需代码**:
```harbour
REQUEST HB_CODEPAGE_UTF8EX

function Main()
   hb_cdpSelect( "UTF8EX" )
   // ...
return nil
```

### 3.2 UTF8EX 的优势
- 提供正确的欧洲语言大小写映射
- 比 `UTF8` 代码页更可靠

### 3.3 字符串字面量
- **普通字符串**: `"abc"`, `'abc'`, `[abc]`
- **转义字符串**: `e"Line1\nLine2\"quoted\""` (支持 C 风格转义)
  - 转义序列: `\n`, `\r`, `\t`, `\\`, `\"`, `\'`, `\xNN`, `\0NNN`

## 4. 条件表达式

### 4.1 优先使用 iif()
- **规则**: 优先使用 `iif()` 进行表达式级条件判断
- **原因**: 避免 `if()` 与语句语法的视觉混淆

**示例**:
```harbour
// 正确
local l_nAverage := iif( l_nCount > 0, l_nSum / l_nCount, 0 )

// 避免（虽然合法）
local l_nAverage := if( l_nCount > 0, l_nSum / l_nCount, 0 )
```

---

## 5. 显式 END 形式

### 5.1 优先使用显式终止符
- **规则**: 优先使用显式终止符而非通用 `end`
- **关键字** (全小写):
  - `endtry`
  - `enddo`
  - `endfor`
  - `endcase`
  - `endwith`
  - `endif`
  - `endswitch`
  - `endwhile`

**示例**:
```harbour
begin sequence
   // ...
recover
   // ...
endtry
```

## 6. 向控制台输出

### 6.1 正确方式
- **规则**: 使用 `OutErr()` 或 `OutStd()` 函数
- **换行符**: 手动添加 `hb_eol()`

**示例**:
```harbour
OutErr( "调试信息", hb_eol() )
OutStd( "程序输出", hb_eol() )
```

### 6.2 注意事项
- GUI 程序中不要使用 `?` 运算符（仅控制台程序使用）
- `OutErr()` 输出到标准错误流
- `OutStd()` 输出到标准输出流

## 7. 数组初始化

### 7.1 避免使用 Array(n, m)
- **问题**: `Array(10, 9)` 可能创建错误结构
- **推荐**: 手动创建二维数组

**示例**:
```harbour
// 推荐方式
aBoardPos := {}
FOR i := 1 TO 10
   aRow := Array(9)
   FOR j := 1 TO 9
      aRow[j] := "0"
   NEXT
   AAdd(aBoardPos, aRow)
NEXT
```

## 8. 常用 Harbour 函数

### 8.1 字符串函数
- `hb_eol()` - 获取操作系统换行符
- `hb_utf8len(cStr)` - 获取 UTF-8 字符串长度
- `hb_strisutf8(cStr)` - 检查是否为 UTF-8 字符串
- `AllTrim(cStr)` - 去除首尾空格
- `Left(cStr, nLen)` - 获取左子串
- `Right(cStr, nLen)` - 获取右子串
- `SubStr(cStr, nStart, nLen)` - 获取子串

### 8.2 类型函数
- `ValType(xValue)` - 获取变量类型
- `Type(cExpression)` - 获取表达式类型
- `hb_IsArray(xValue)` - 检查是否为数组
- `hb_IsHash(xValue)` - 检查是否为哈希
- `hb_IsObject(xValue)` - 检查是否为对象

### 8.3 数组函数
- `AAdd(aArray, xValue)` - 添加元素
- `AIns(aArray, nPos)` - 插入元素
- `ADel(aArray, nPos)` - 删除元素
- `ASize(aArray, nSize)` - 调整数组大小
- `AClone(aArray)` - 深拷贝数组
- `ACopy(aSource, aTarget)` - 拷贝数组
- `Len(aArray)` - 获取数组长度

### 8.4 哈希函数
- `hb_Hash()` - 创建空哈希
- `hb_HGet(hHash, cKey)` - 获取值
- `hb_HSet(hHash, cKey, xValue)` - 设置值
- `hb_HDel(hHash, cKey)` - 删除键
- `hb_HKeys(hHash)` - 获取所有键
- `hb_HValues(hHash)` - 获取所有值

### 8.5 其他函数
- `iif(lCondition, xTrue, xFalse)` - 条件表达式
- `Empty(xValue)` - 检查是否为空
- `NIL` - 空值（不是 .f.）
- `hb_DateTime()` - 获取当前时间戳
- `Seconds()` - 获取秒数

## 11. 文件编辑规范

### 11.1 编辑现有文件
- **规则**: 始终使用 `replace` 工具，不要使用 `write_file`
- **流程**:
  1. 使用 `read_file` 确认内容
  2. 使用 `replace` 进行小修改
  3. 避免一次性大替换

### 11.2 函数签名修改
- 修改函数签名时，务必同步更新头文件（`.ch`）和所有调用处
- `xq_IsMoveCorrect` 是核心函数，被多处调用，修改需谨慎
