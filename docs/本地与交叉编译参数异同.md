# 本地与交叉编译参数异同

本文档记录中国象棋项目使用 mingw64（ucrt64）编译器时，本地 Linux 构建与 Windows 交叉编译的参数差异。

## 四种构建目标

| 构建目标 | 配置文件 | 输出文件 | 编译器 |
|----------|----------|----------|--------|
| Linux GUI | xq.hbp | xq | gcc |
| Linux Console | xq_con.hbp | xq_con | gcc |
| Windows GUI | xq.hbp | xq.exe | x86_64-w64-mingw32ucrt-gcc |
| Windows Console | xq_con.hbp | xq_con.exe | x86_64-w64-mingw32ucrt-gcc |

---

## 编译器信息

### Linux 本地编译器
- **编译器**: gcc
- **版本**: 14.2.0
- **架构**: x86_64-linux-gnu

### Windows 交叉编译器
- **编译器**: x86_64-w64-mingw32ucrt-gcc
- **版本**: 14.2.0
- **架构**: x86_64-windows-gnu (UCRT)
- **特性**: 使用 Microsoft Universal C Runtime (UCRT)

---

## 编译参数对比

### 1. Linux GUI (xq.hbp)

**编译参数：**
```bash
gcc -c -g -W -Wall -pedantic -pipe \
  -DHWG_USE_POINTER_ITEM \
  -I/opt/harbour/include \
  -I/opt/hwgui/include \
  xiangqiw.c -o xiangqiw.o
```

**链接参数：**
```bash
gcc xiangqiw.o ... \
  -Wl,--start-group \
  -lhwgui -lprocmisc -lhbxml \
  -lgdiplus \
  -lhbextern -lhbdebug -lhbvm -lhbrtl -lhblang -lhbcpage \
  -lgtcgi -lgtpca -lgtstd -lgtwin -lgtwvt -lgtgui \
  -lhbrdd -lhbuddall -lhbusrrdd -lrddntx -lrddcdx -lrddnsx -lrddfpt \
  -lhbrdd -lhbhsx -lhbsix -lhbmacro -lhbcplr -lhbpp -lhbcommon -lhbmainstd \
  -Wl,--end-group \
  `pkg-config --libs gtk+-2.0` \
  -oxq
```

**特点：**
- 本地编译，无需 target 参数
- 使用 GTK+ 2.0 GUI 库
- 包含路径为本地 HwGUI 头文件
- 库路径为本地 Linux 库
- 使用 pkg-config 获取 GTK+ 库和编译选项

---

### 2. Linux Console (xq_con.hbp)

**编译参数：**
```bash
gcc -c -g -W -Wall -pedantic -pipe \
  -I/opt/harbour/include \
  xiangqi.c -o xiangqi.o
```

**链接参数：**
```bash
gcc xiangqi.o ... \
  -Wl,--start-group \
  -lhbextern -lhbdebug -lhbvm -lhbrtl -lhblang -lhbcpage \
  -lgtcgi -lgtpca -lgtstd -lgtwin -lgtwvt -lgtgui \
  -lhbrdd -lhbuddall -lhbusrrdd -lrddntx -lrddcdx -lrddnsx -lrddfpt \
  -lhbrdd -lhbhsx -lhbsix -lhbmacro -lhbcplr -lhbpp -lhbcommon -lhbmainstd \
  -Wl,--end-group \
  -oxq_con
```

**特点：**
- 无 GUI 定义宏 (`-DHWG_USE_POINTER_ITEM`)
- 无 HwGUI 库链接
- 无 GTK+ 依赖
- 最简洁的配置

### 3. Windows GUI (xq.hbp)

**编译参数：**
```bash
x86_64-w64-mingw32ucrt-gcc -c \
  -D__USE_MINGW_ANSI_STDIO=0 \
  -DHWG_USE_POINTER_ITEM \
  -g -W -Wall -pedantic -pipe \
  -DUNICODE -D_UNICODE \
  -I/opt/harbour/include \
  -I/opt/hwgui/include \
  xiangqiw.c -o xiangqiw.o
```

**链接参数：**
```bash
x86_64-w64-mingw32ucrt-gcc xiangqiw.o ... \
  -Wl,--subsystem,windows \
  -Wl,--entry=mainCRTStartup \
  -Wl,--start-group \
  -lhwgui -lprocmisc -lhbxml \
  -lgdiplus \
  -lhbextern -lhbdebug -lhbvm -lhbrtl -lhblang -lhbcpage \
  -lgtcgi -lgtpca -lgtstd -lgtwin -lgtwvt -lgtgui \
  -lhbrdd -lhbuddall -lhbusrrdd -lrddntx -lrddcdx -lrddnsx -lrddfpt \
  -lhbrdd -lhbhsx -lhbsix -lhbmacro -lhbcplr -lhbpp -lhbcommon -lhbmainstd \
  -lwinmm -lkernel32 -luser32 -lgdi32 -ladvapi32 -lws2_32 -liphlpapi \
  -lwinspool -lcomctl32 -lcomdlg32 -lshell32 -luuid -lole32 -loleaut32 \
  -lmpr -lmapi32 -limm32 -lmsimg32 -lwininet \
  -lhbpcre -lhbzlib \
  -Wl,--end-group \
  -L/opt/harbour/lib/win/mingw64 \
  -L/opt/hwgui/lib/win/mingw64 \
  -oxq.exe
```

**特点：**
- **关键参数：** `x86_64-w64-mingw32ucrt-gcc`（使用 ucrt64 交叉编译器）
- **关键参数：** `-D__USE_MINGW_ANSI_STDIO=0`（ucrt64 编译器需要）
- UNICODE 宏定义：`-DUNICODE -D_UNICODE`
- Windows 子系统：`-Wl,--subsystem,windows`（替代 -mwindows）
- 入口点：`-Wl,--entry=mainCRTStartup`（GUI 应用程序）
- GDI+ 图形库：`-lgdiplus`
- Windows 系统库：kernel32, user32, gdi32, winmm 等

**ucrt64 特殊处理：**
- ucrt64 编译器不支持 `-mwindows` 和 `-mconsole` 选项
- 需要使用包装脚本移除 Harbour 自动添加的 `-mconsole` 选项
- 使用 `-Wl,--subsystem,windows` 替代 `-mwindows`
- 使用 `-Wl,--entry=mainCRTStartup` 指定 GUI 应用程序入口点

## 核心区别总结

### Windows 交叉编译必须参数
1. **编译器：** `x86_64-w64-mingw32ucrt-gcc`
2. **`-D__USE_MINGW_ANSI_STDIO=0`** - ucrt64 编译器必须参数
3. **`-DUNICODE -D_UNICODE`** - Windows UNICODE 支持

### Windows 子系统区分
1. **GUI 应用：** `-Wl,--subsystem,windows -Wl,--entry=mainCRTStartup`
2. **控制台应用：** 无特殊选项（默认为 console 子系统）

### GUI 库差异
| 平台 | GUI 库 |
|------|--------|
| Linux | GTK+ 2.0 |
| Windows | Win32 API + GDI+ |

### ucrt64 vs 传统 mingw64
| 特性 | 传统 mingw64 | ucrt64 |
|------|-------------|--------|
| 编译器名称 | x86_64-w64-mingw32-gcc | x86_64-w64-mingw32ucrt-gcc |
| 运行时 | MSVCRT (旧版) | UCRT (新版，推荐) |
| `-mwindows` | 支持 | ❌ 不支持 |
| `-mconsole` | 支持 | ❌ 不支持 |
| `-target` 选项 | 不支持 | ❌ 不支持 |

## 构建脚本 (build.sh)

### 关键函数

#### check_mingw64()
检查 mingw64 编译器是否可用（支持传统 mingw64 和 ucrt64）：
```bash
check_mingw64() {
    # 检查传统的 mingw64 编译器
    if command -v x86_64-w64-mingw32-gcc &> /dev/null; then
        return 0
    fi
    # 检查 ucrt64 编译器
    if command -v x86_64-w64-mingw32ucrt-gcc &> /dev/null; then
        return 0
    fi
    return 1
}
```

#### build_gui() - ucrt64 特殊处理
为 ucrt64 编译器创建临时包装脚本，移除不支持的选项：
```bash
if [ "$use_ucrt64" = "1" ]; then
    local wrapper_script="/tmp/hbmk_gcc_wrapper_$$"
    cat > "$wrapper_script" << 'EOF'
#!/bin/bash
# 移除 -mconsole 选项（ucrt64 编译器不支持）
filtered_args=()
for arg in "$@"; do
    if [ "$arg" != "-mconsole" ]; then
        filtered_args+=("$arg")
    fi
done
exec x86_64-w64-mingw32ucrt-gcc "${filtered_args[@]}"
EOF
    chmod +x "$wrapper_script"
    
    # 创建临时目录，包含包装脚本
    local tmpdir="/tmp/hbmk_wrapper_$$"
    mkdir -p "$tmpdir"
    cp "$wrapper_script" "$tmpdir/gcc"
    chmod +x "$tmpdir/gcc"
    
    # 临时设置 PATH 以使用包装脚本
    export PATH="$tmpdir:$PATH"
fi
```

## 注意事项

1. **ucrt64 vs 传统 mingw64**
   - ucrt64 是新版运行时，兼容性更好
   - 传统 mingw64 已在 Debian 13 中标记为 rc（removed candidate）
   - 优先使用 ucrt64 编译器

2. **子系统选项**
   - ucrt64 不支持 `-mwindows` 和 `-mconsole`
   - 必须使用 `-Wl,--subsystem,windows` 替代 `-mwindows`
   - 控制台应用无需特殊选项（默认为 console）

3. **包装脚本**
   - build.sh 会自动为 ucrt64 创建包装脚本
   - 包装脚本移除 Harbour 自动添加的 `-mconsole` 选项
   - 临时脚本会在构建完成后自动清理

4. **库路径**
   - 所有路径配置在 `xq.hbp` 和 `xq.hbc` 中
   - `build.sh` 中的路径变量仅用于信息显示
   - 不要在 build.sh 中重复传递路径参数

## 编译输出

### Linux GUI
- **文件名：** `xq`
- **类型：** ELF 64-bit LSB executable, x86-64
- **大小：** 约 5MB

### Linux Console
- **文件名：** `xq_con`
- **类型：** ELF 64-bit LSB executable, x86-64
- **大小：** 约 2MB

### Windows GUI
- **文件名：** `xq.exe`
- **类型：** PE32+ executable for MS Windows 5.02 (GUI), x86-64
- **大小：** 约 3.2MB
- **子系统：** Windows GUI

### Windows Console
- **文件名：** `xq_con.exe`
- **类型：** PE32+ executable for MS Windows 5.02 (console), x86-64
- **大小：** 约 2.5MB
- **子系统：** Windows Console
