# 中国象棋 GUI 项目 - 完整设计文档

## 项目概述

本项目是一个使用 Harbour 语言和 HwGUI 库开发的中国象棋桌面应用程序，具有图形用户界面和内置 AI 引擎。

**项目类型**: GUI 桌面应用程序（象棋游戏引擎）
**开发语言**: Harbour 3.2.0dev
**GUI 框架**: HwGUI
**构建工具**: hbmk2 (位于 `/opt/harbour/bin/hbmk2`)
**后端编译器**: GCC (Linux), Mingw64 (Windows 交叉编译)
**版本**: 1.0.0 (2026-03-01)
**参考架构**: 基于 sunfish_prg 设计

## 核心数据结构

### 1. 棋盘表示

**存储格式**: 90 字符字符串（10行×9列）
- 索引范围: 1-90（Harbour 字符串从 1 开始）
- 计算方式: `index = row * 9 + col + 1`（row, col 为 0-based）

**棋子编码**（单字符代码）:
- 红方（大写）: K(帅), A(仕), B(相), N(马), R(车), C(炮), P(兵)
- 黑方（小写）: k(将), a(士), b(象), n(马), r(车), c(炮), p(卒)
- 空位: '0'（数字0）

**示例初始棋盘**:
```
rnbakabnr  (行0 - 黑方底线)
000000000
0c00000c0
p0p0p0p0p
000000000  (河界上沿)
000000000  (河界下沿)
P0P0P0P0P
0C00000C0
000000000
RNBAKABNR  (行9 - 红方底线)
```

**棋子代码对照表**:

| 棋子（红/黑） | FEN 代码 | WXF 英译 | 中文（红/黑） | 说明 |
|-------------|----------|----------|--------------|------|
| 帅/将 | K/k | K | 帥/將 | King |
| 仕/士 | A/a | A | 仕/士 | Advisor |
| 相/象 | B/b | E | 相/象 | Elephant |
| 傌/馬 | N/n | H | 傌/馬 | Horse |
| 俥/車 | R/r | R | 俥/車 | Rook |
| 炮/砲 | C/c | C | 炮/砲 | Cannon |
| 兵/卒 | P/p | P | 兵/卒 | Pawn |

**代码说明**:
- **FEN 代码**: 国际象棋标准表示法，用于 FEN 字符串和棋盘数组
  - 红方使用大写字母（K, A, B, N, R, C, P）
  - 黑方使用小写字母（k, a, b, n, r, c, p）
- **WXF 英译**: 世界象棋联合会（WXF）标准英译代码，用于英文记谱法
  - 注意：象/相的 WXF 代码是 E（Elephant），而非 B
- **中文**: 传统中国象棋汉字，用于中文记谱法和界面显示

### 2. 位置结构

核心引擎使用 10 元素数组表示棋局状态：

```prg
aPos := Array(10)
aPos[1]    := cBoard      // 90字符棋盘字符串
aPos[2]    := nR0         // 上一次移动（中国象棋未使用）
aPos[3]    := nRedKing    // 红方帅位置索引
aPos[4]    := nBlackKing  // 黑方将位置索引
aPos[5]    := cPrevMove   // 前一步移动（用于检查重复）
aPos[6-9]  := ...         // 保留字段
aPos[10]   := lTurn       // 当前回合（.T.=红方, .F.=黑方）
```

### 3. 移动编码

**格式**: `nMove = nFrom + nTo * 256`
- `nFrom`: 源位置索引（0-89）
- `nTo`: 目标位置索引（0-89）

**示例**: 从位置 0 移动到 89 → `0 + 89 * 256 = 22784`

### 4. 坐标系统

**GUI/TUI 坐标**（用户界面）:
- 行: 1-10（从上到下）
  - 行 1 = 黑方底线
  - 行 10 = 红方底线
- 列: 1-9（从左到右）
- 用途: 用户点击屏幕/终端、显示棋子位置

**UCCI 坐标**（引擎和内部逻辑）:
- 行: 0-9（从下到上）
  - 行 0 = 红方底线
  - 行 9 = 黑方底线
- 列: 0-8（从左到右）
  - 列 0 = 左侧 (a)
  - 列 8 = 右侧 (i)
- 格式: 列字母 + 行数字（如 "a0"、"e9"）
- 用途: UCCI 协议、移动验证、引擎内部逻辑

**数组索引**（Harbour 1-based）:
- 范围: 1-90
- XQP/FEN 顺序: 从黑方底线到红方底线
  - 索引 1-9: 黑方底线（GUI行1，UCCI行9）
  - 索引 82-90: 红方底线（GUI行10，UCCI行0）
- 用途: 棋盘数组存储

**坐标转换关系**:
- GUI 行 = 10 - UCCI 行
- GUI 列 = UCCI 列 + 1
- 数组索引 = (GUI 行 - 1) * 9 + GUI 列
- 数组索引 = (9 - UCCI 行) * 9 + (UCCI 列 + 1)

**坐标转换函数**:
```prg
// GUI/TUI → UCCI
aUCCI := xq_GUIToUCCI( nGUIRow, nGUICol )

// UCCI → GUI/TUI
aGUI := xq_UCCIToGUI( nUCCIRow, nUCCICol )

// UCCI → 数组索引
nIdx := xq_UCCIToIdx( nUCCIRow, nUCCICol )

// 数组索引 → UCCI
aUCCI := xq_IdxToUCCI( nIdx )

// UCCI → UCCI 字符串（如 "a0"）
cUCCI := xq_UCCIToStr( nUCCIRow, nUCCICol )

// UCCI 字符串 → UCCI
aUCCI := xq_StrToUCCI( cUCCICoord )

// GUI/TUI → 数组索引
nIdx := xq_GUIToIdx( nGUIRow, nGUICol )

// 数组索引 → GUI/TUI
aGUI := xq_IdxToGUI( nIdx )
```

**坐标映射示例**:
- GUI(1,1)  黑方左侧车  → UCCI(0,9) → 数组索引1   → UCCI字符串 "a9"
- GUI(1,5)  黑方将      → UCCI(4,9) → 数组索引5   → UCCI字符串 "e9"
- GUI(10,5) 红方帅      → UCCI(4,0) → 数组索引86  → UCCI字符串 "e0"
- GUI(10,1) 红方左侧车  → UCCI(0,0) → 数组索引82  → UCCI字符串 "a0"

## 中国象棋规则实现

### 1. 宫殿规则

```
红方宫殿: 行 7-9, 列 3-5
黑方宫殿: 行 0-2, 列 3-5

将/帅、士只能在宫殿内移动
```

**实现函数**:
- `xq_IsInRedPalace(nRow, nCol)` - 检查是否在红方宫殿
- `xq_IsInBlackPalace(nRow, nCol)` - 检查是否在黑方宫殿

### 2. 河界规则

```
河界: 行 4-5 之间

相/象不能过河
兵/卒过河后可以横走
```

**实现逻辑**:
- 红兵过河判断: `nLogicalRowFrom >= 5`
- 黑卒过河判断: `nLogicalRowFrom <= 4`

### 3. 别马腿

```
马走"日"字，但马腿位置有子则不能跳

马腿位置: 从起点到终点的中间位置
```

**实现函数**:
- `xq_CheckHorseLeg(aBoard, nFromRow, nFromCol, nToRow, nToCol)`

### 4. 塞象眼

```
象走"田"字，但象眼位置有子则不能飞

象眼位置: 两条对角线的交点
```

**实现函数**:
- `xq_CheckElephantEye(aBoard, nFromRow, nFromCol, nToRow, nToCol)`

### 5. 炮打隔山

```
炮移动: 直线移动，中间不能有子
炮吃子: 直线移动，中间必须恰好有一个子（炮架）
```

**实现函数**:
- `xq_CheckCannonMove(aBoard, nFromRow, nFromCol, nToRow, nToCol)`

### 6. 白脸将

```
双方将/帅不能直接面对面（中间无子）
```

**实现函数**:
- `xq_CheckKingsFacing(aBoard)` - 白脸将检测

### 7. 将军、将死和困毙

```
将军: 将/帅被对方棋子攻击
将死: 将军且无合法走法
困毙: 未被将军但无合法走法
```

**实现函数**:
- `xq_IsKingInCheck(aBoard, lRed)` - 将军检测
- `xq_IsCheckmate(aBoard, lRed)` - 将死检测
- `xq_HasLegalMoves(aBoard, lRed)` - 检查是否有合法走法（用于困毙检测）
- `CheckGameState(aBoard, lRedTurn)` - 检查游戏状态（将死/困毙）

## 编程规范

### 变量命名（匈牙利命名法）
- `oObject` - 对象
- `cString` - 字符串
- `nNumber` - 数字
- `lLogical` - 逻辑值
- `aArray` - 数组

### 变量作用域
- `LOCAL` - 局部变量（函数作用域）
- `STATIC` - 静态变量（初始化一次，保持值）
- `PRIVATE` - 私有变量（函数及子函数作用域）
- `PUBLIC` - 全局变量（整个应用程序）

### 代码风格
- 遵循 Harbour 语法约定
- 使用适当的变量作用域
- 包含函数文档注释
- 与现有代码模式保持一致

### 注释要求
- 为关键函数添加详细的参数和返回值说明
- 在复杂逻辑处添加注释说明
- 不要通过注释与用户对话

### 错误处理
- 使用 `BEGIN SEQUENCE` / `RECOVER` / `END SEQUENCE`
- 记录错误到日志
- 通过 `hwg_MsgStop()` 或 `hwg_MsgInfo()` 提供用户友好消息

## 开发注意事项

### 文件编辑
1. **编辑现有文件**: 始终使用 `replace` 工具，不要使用 `write_file`
2. **修改前确认**: 使用 `replace` 前先用 `read_file` 确认内容
3. **小修改**: 分多次小修改，避免一次性大替换

### 编译调试
1. **每次修改后构建**: 修改代码后必须运行 `~/hwbuild/hwbc xiangqi.hwprj`
2. **启用跟踪**: 在 Main() 函数开头取消注释
   ```prg
   hb_traceState( 1 )
   hb_traceLevel( 5 )
   ```
3. **编译警告**: 警告可能指示潜在问题，但少量警告不影响功能

### 函数签名
- 修改函数签名时，务必同步更新头文件（`.ch`）和所有调用处
- `xq_IsMoveCorrect` 是核心函数，被多处调用，修改需谨慎

### 数组初始化
- Harbour 的 `Array(10, 9)` 可能创建错误结构
- 推荐手动创建：
  ```prg
  aBoardPos := {}
  FOR i := 1 TO 10
      aRow := Array(9)
      FOR j := 1 TO 9
          aRow[j] := "0"
      NEXT
      AAdd(aBoardPos, aRow)
  NEXT
  ```

### GUI 与引擎同步
- `aBoardPos` 是 GUI 使用的二维数组
- `aPos[POS_BOARD]` 是引擎使用的字符串
- 移动后需同步更新两者
